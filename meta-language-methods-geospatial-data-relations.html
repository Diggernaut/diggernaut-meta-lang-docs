Relations

A relation is a collection of points, lines, and other relations that are united into a single object. For example, the relation could be: country, region, region, city, street, etc. Relations have a different administrative level. For example, level 2 corresponds to the country, other levels vary depending on the country. More detailed information about administrative levels in different countries can be found here.

To work with relations there is a command relation. It allows you to search for a relation by name, ID, administrative level and the country. Thus, you can get, for example, all the cities or regions in a certain country, or find a certain city.

The following parameters are used in the command:

Parameter Description
country_name English name of the country where you want to search for a relation. Optional parameter.
admin_level Administrative level. Optional parameter.
name English relation name, eg name of city. Optional parameter.
id Relation ID. If used, system will search by given ID regardless of other parameters sent. Optional parameter.
